# RRWeb React Kit 测试方案

## 1. 测试目标
确保录制全过程无遗漏，回放过程平滑，UI 状态同步准确。

## 2. 人工测试场景
### 2.1 录制功能测试
- [ ] 点击 "开始录制" 后，确认为录制状态。
- [ ] 在交互区输入文字、点击按钮、悬停。
- [ ] 点击 "停止录制" 后，确认事件列表长度增加。

### 2.2 回放控制测试
- [ ] 播放、暂停功能是否正常。
- [ ] 重置功能：点击重置是否能回到第 0 秒。
- [ ] 倍速切换：分别在播放中和暂停时切换 1x, 2x, 4x，确认回放速度有明显变化。
- [ ] 边界测试：录制 1 秒和录制 30 秒的情况下，播放进度的准确性。

### 2.3 UI/UX 测试
- [ ] 弹窗测试：录制过程中触发 Modal 弹窗，回放时弹窗是否能正常居中显示。
- [ ] 缩放测试：在不同分辨率的浏览器窗口下查看播放器器是否自适应或显示滚动条。

## 3. 自动化测试 (规划中)
未来将引入 Playwright 进行端到端 (E2E) 测试：
- 模拟用户交互。
- 对比录制后的 JSON 事件序列与预期序列。
- 自动化快照对比回放时的 DOM 状态。
